# ROLE 및 SCHEMA 백업과 복구
PostgreSQL의 유저는 ROLE(계정)과 SCHEMA(객체)로 분리되어 있으므로 완벽한 복구를 위해서는 계정 정보와 데이터를 모두 백업하고 복구해야 한다

## 환경 설정 및 백업 실행
- 먼저 관리자 권한으로 접속하여 백업 파일을 생성합니다.

```Bash
# 1. postgres 유저로 전환 및 환경 변수 정리
[root@localhost ~]# su - postgres
-bash-4.2$ unset PGHOST PGPORT PGUSER PGDATABASE

# 2. 스키마 데이터 백업 (Custom Format)
-bash-4.2$ pg_dump -U postgres -h localhost -d testdb -n scott -Fc -f scott_schema.dump

# 3. 롤(계정) 정보만 백업 (Plain Text SQL)
-bash-4.2$ pg_dumpall -U postgres -h localhost --roles-only -f roles.sql

# 4. 백업 파일 확인
-bash-4.2$ ls -l scott_schema.dump roles.sql
-bash-4.2$ pg_restore -l scott_schema.dump | head -20
```

## 기존 데이터 및 롤 완전 삭제 (Clean Up)
- 복구 테스트를 위해 기존의 스키마와 롤을 삭제한다
- 롤 삭제 시 발생하는 의존성 오류 해결이 핵심이다
```bash
-- 1. scott 스키마 삭제
testdb=# DROP SCHEMA scott CASCADE;

-- 2. 롤 삭제 시도 (의존성 오류 발생 시 처리)
testdb=# ALTER DATABASE testdb OWNER TO postgres;
testdb=# REASSIGN OWNED BY scott TO postgres;
testdb=# DROP OWNED BY scott;

-- 3. 다른 DB(postgres)에 남은 의존성 제거
testdb=# \c postgres
postgres=# REASSIGN OWNED BY scott TO postgres;
postgres=# DROP OWNED BY scott;
postgres=# DROP ROLE scott;
```

## 데이터 복구 (Restore)
- 백업해둔 파일들을 이용해 롤을 먼저 살리고 그 다음 데이터를 복구한다

```Bash
# 1. 롤(계정) 정보 복구
-bash-4.2$ psql -U postgres -h localhost -d postgres -f roles.sql

# 2. 스키마 및 테이블 데이터 복구
-bash-4.2$ pg_restore -U postgres -h localhost -d testdb scott_schema.dump
4.4 복구 결과 확인
데이터가 정상적으로 들어왔는지 scott 계정으로 접속이 가능한지 확인
```

```bash
-- 1. 스키마 및 테이블 목록 조회
testdb=# \dn
testdb=# \dt scott.*

-- 2. 데이터 건수 확인
testdb=# select count(*) from scott.emp;
 count 
-------
    14
(1개 행)

-- 3. scott 계정으로 접속 테스트
-bash-4.2$ psql -U scott -h localhost -d testdb
testdb=> select ename, sal from scott.emp limit 3;
 ename |   sal   
-------+---------
 SMITH |  800.00
 ALLEN | 1600.00
 WARD  | 1250.00
 ```


ROLE 백업 필수: pg_dump는 계정 생성 구문을 포함하지 않고 pg_dumpall --roles-only를 병행해야 한다

REASSIGN OWNED: 사용자를 지울 수 없을 때는 모든 DB에 접속하여 해당 명령어로 소유권을 이전해야 한다

Search Path: 복구 후 테이블이 안 보인다면 SET search_path TO scott;을 실행
